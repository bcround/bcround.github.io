TIL 20/10/25 일요일 자습

오늘 한 일

오늘은 처음으로 주말에 패스트캠퍼스 학원에 나와서 자습을 했다. 금요일날 강의에서 다루었던 HTML구조와 CSS 디자인을 복습하고 다시 처음부터 만들어 보았고 개념이 부족한 부분들에 대해서 동영상 강의를 다시 보았다. 강사님과 한 방법과 다르게 해보다보니 문제점들이 발생하였고 그 문제점들을 해결하는 과정에서 새로운 정보들을 많이 알게 되었다. 게으르게 행동하지 말고 매주 주말마다 적어도 한번은 학원에 나와서 자습을 해야겠다.

radial-gradient

  • radial-gradient 속성은 원형으로 그라데이션 효과를 주기 위해서 사용한다. CSS3에서 추가되었고 IE10이상에서 지원한다.
  • 문법
    • background: radial-gradient({shape} {size} at {position}, color1, color2, ...)
      • shape: 원인지 타원형 모양인지를 정해준다. (circle or ellipse)
      • size: 크기를 정해준다.
      • position: 중심의 위치를 정한다.
      • color: 색을 정한다. 색 뒤에 길이를 넣어서 색이 차지하는 범위를 정할 수 있다. (% or px)

inline-block 요소의 baseline

  • text가 없는 비어있는 inline-block의 경우:

    • inline-block 요소의 baseline은 margin box의 하단 부분이 된다.

  • text가 있는 inline-block의 경우:

    • normal flow 에서 마지막 text content요소가 baseline이 된다. 즉, text의 마지막줄이 baseline이 된다.

  • overflow의 값이 visible이 아닌 다른 값의 경우

    • margin box의 하단부분이 baseline으로 잡힌다.

TIL 20/10/23 HTML/CSS 다섯번째 강의

내가 오늘 배우고 느낀점

오늘은 웹카페 클론 실습에서 main쪽 영역에 로그인창 CSS 디자인과 그 밑에 링크들 그리고 웹 관련 용어 섹션에 대해 배웠다. 로그인창 CSS 설계만하고 어제 강의를 마쳤는데 어젯밤에 오늘 수업에서 했던 CSS디자인을 혼자 해보았다. 강사님이 가르쳐주신것과 내가 짠 코드를 비교하면서 들었는데 새로 알게 된 부분이 많았고 코드를 좀 더 간단하게 짜는 방법을 배웠다. 오늘도 배웠던 내용을 복습하고 다시 처음부터 혼자서 스스로 코드를 짜봐야겠다.

CSS Preprocessors

  • CSS 전처리기는 자신만의 유니크한 syntax로 CSS를 생성하는 프로그램이다.
  • 프로젝트의 규모가 커짐에 따라 웹페이지도 커지고 복잡해짐에 따라 스타일도 많이사용하고 불필요한 선택자들의 사용으로 코드가 복잡해질 수 있고 그로인해 유지보수가 어려워질 수 있다. 이때 이러한 불편한점과 한계를 보완하기위해 등장한게 CSS 전처리기이다.
  • CSS 전처리기에는 다음과 같은 종류들이 있다.
    • Sass
    • Less
    • Stylus

ETC

  • XHTML에서는 <form></form>안에 <fieldset></fieldset>과 <legend></legend>가 필수. HTML 4.01,5에서는 필수는 아님.
  • 접근성 차원에서 <input>같은 입력서식은 <label></label>을 무조건 1:1 대응해서 사용한다.
  • 글로벌어트리뷰트(전역속성)
    • id
    • class
    • lang
    • data-*
    • style
    • title
  • em - 현재글자크기 비례. 사이즈 단위
  • text-align은 block요소에 적용해야한다. 그러면 그안에 inline요소들이 정렬된다.
  • vertical-align은 inline 요소에 적용해야된다.
  • <button></button>을 사용할때는 padding값을 재지정해주는게 좋다. 브라우저마다 padding값이 다르기 때문이다.
  • HTML 5.2이상 부터는 <dl></dl>태그 안에도 <div></div>태그 사용이 가능하다. 그러나 <dt></dt>하나의 요소만 <div></div>로 묶는건 불가능하고 <dt><dd></dd></dt> 전체를 묶을때만 사용할 수 있다. 위와 같은 방법은 사용할 수 없다.
    <div></div>를 사용하려면 아래와 같이 사용해야된다. <dl></dl>내의 <div></div>에는 <dt></dt>과 <dl></dl>외에는 다른요소가 들어올 수 없다.

틀리고 배운점

  • <input>이나 <button></button> 요소를 사용하면서 type 어트리뷰트 값으로 submit을 선언하면 브라우저는 어트리뷰트로 설정된 값을 가지고 서버에 request한다.

TIL 20/10/21 HTML/CSS 네번째 강의

내가 오늘 배우고 느낀점

오늘은 웹카페 클론 실습중 visual에 보여지는 애니메이션 부분과 메인섹션 부분에 로그인창 html구조 짜는법 그리고 로그인창 CSS 설계에 대해서 배웠다. animation의 속성중 모르는 것들을 많이 배웠고 form과 그안에 fieldset, legend에 대해서도 배웠다. 공부할 시간이 모자라서 오늘 아침부터 한시간씩 일찍 오기 시작했다. 그전날 수업에서 해봤던 내용들을 다시 처음부터 html설계와 css입히는 연습을 해야겠다는 필요성을 느껴서이다. TIL도 중요하지만 코드를 복습하고 다시 적용해보는 연습을 더해야겠다.

Animation

  • animation은 요소에 적용되는 CSS 스타일을 다른 CSS 스타일로 부드럽게 적용 (전환)시켜준다.
  • animation의 속성은 다음과 같다.
    • @keyframes : animation이 어떻게 돌아갈지 코드를 명시하는 부분이다.
    • animation : animation의 단축속성이다.
    • animation-delay : animation의 요소가 불러와지고 언제 시작할지 정한다.
    • animation-direction : animation이 끝나고 처음부터 다시 시작할지 역방향으로 시작할지 정한다.
    • animation-duration : animation이 완전한 한 싸이클을 도는 시간을 명시해준다.
    • animation-fill-mode : animation이 실행 전과 후에 대상에 스타일을 지정하는 방식을 설정한다. 속성값이 forwards면 @keyframes에 설정된 마지막 값을 유지한다. backwards는 @keyframes의 설정된 초기값을 유지한다.
    • animation-iteration-count : animation이 몇 번 실행되어야하는지 명시해준다. 속성값으로 infinite을 설정해주면 무한 반복된다.
    • animation-name : animation의 중간상태를 지정해준다. 중간상태는 @keyframes을 사용하여 설정한다.
    • animation-play-state : animation을 멈추거나 다시시작할 수 있다.
    • animation-timing-function : animation의 전환을 어떤 시간 간격으로 움직일지 지정한다.
  • animation-name과 animation-duration은 필수 속성 이다.

  • <form></form> 태그는 사용자로 부터 입력받는 입력양식을 감싸는 태그이다.
    • ex:
  • <form></form>은 입력된 데이터를 한번에 서버로 전송하는 역할을 한다.
  • <form></form> 자체에다가는 css디자인을 잘 하지 않는다.

fieldset

  • <fieldset></fieldset>는 사용자 입력을 받는 <form></form>안의 컨트롤들을 하나의 그룹으로 묶는 역할을 한다.
    • ex:

legend

  • <fieldset></fieldset>의 제목역할을 한다. <fieldset></fieldset> 묶음의 성격 용도등을 나타내기도 한다.

ETC

  • 요소 박스내에서는 ::before와 ::after는 하나밖에 사용못한다. (before의 before는 못만들어냄)
  • html에서 &표시를 표현하고 싶으면 &amp;
  • 투명도를 정할때 opacity를 사용하면 배경값까지 다 흐리게 만들어진다. 글자만 흐리게하려면 color: rgba();를 사용하자.
  • 요소를 이동하는 방법 4가지:
    • transform: translate();
    • margin: ;
    • padding: ;
    • position: ;
  • background-image는 먼저 만든 이미지가 위에 놓인다.
  • id값을 네이밍할때는 camelCase로 하자.
  • text-indent는 inline요소에서 사용하지 못한다.

틀리고 배운점

  • position: absolute; 로 선언된 요소의 display 프로퍼티 값은 block이 된다.
  • position: absolute; 로 선언된 요소의 위치를 offset으로 설정할 때, offset 값의 기준은 해당 요소의 가장 가까운 상위 요소 중 position 프로퍼티 값이 relative 뿐만 아니라 fixed, absolute, sticky로 선언된 요소이다.
  • 현재 위치한 디렉토리를 원격 저장소(origin)의 브랜치(main)에 연결하는 깃 명령어:

TIL 20/10/21 HTML/CSS 세번째 강의

내가 오늘 배우고 느낀점

오늘은 패스트캠퍼스에서 html/css 세번째 강의 시간 이였다. 오늘은 웹카페 클론 실습중 header에 navigation에 메인메뉴를 만들어 정렬하는 것을 배웠다. inline-block, block, flex에 대해도 배웠는데 아직 개념정리가 잘 안된 것 같아 더 구글링해서 찾아봐야겠다. 그리고 주말에 사전온라인강의도 다시들어보고 노트테이킹을 열심히 해놔서 개념을 머리 속에 굳혀놔야겠다.

button & a tag

  • <button></button>은 inline block
  • <button></button>의 box-sizing의 default값은 border-box이다.
  • <button></button>이 눌려졌는지 안눌렸는지 장애인들을 위한 알려주는 속성은 aria-pressed, 팝업이 있는지 없는지 aria-popup, 팝업이 펼쳐졌는지 안펼쳐졌는지는 aria-expanded
  • <button></button>의 default type은 submit이다.
  • <a></a> tag의 box-sizing의 default 값은 content-box이다.
  • <a></a> tag를 버튼으로 사용하려면 role="button"이라는 프로퍼티를 꼭 넣어주자. 왜냐하면 일반적으로 <a></a> tag는 클릭시 어디로 이동할거라고 예상하는데 버튼을 명시해줌으로써 클릭됬을때 이벤트가 일어날거라고 알려줄 수 있기 때문.

TIL 20/10/20 HTML/CSS 두번째 강의

내가 오늘 배우고 느낀점

오늘은 두번째 강의 였다. 강의에서 float, display:flex, div같은 block요소들 가운데로 센터 시키는법, float를 해제 시키는 방법, position등에 대해 배웠다.
온라인 강의를 들었을때 몰랐거나 애매했던 부분들도 어느정도 해소가 되었고 아직도 모르는 부분은 구글링을 해서 더 찾아봐야겠다. 모르는 부분이 있다고 포기하고 좌절하지말고 좀더 검색하는 습관을 키워야겠다.

display: flex

  • flex는 모던 브라우저에서만 지원한다. flex MDN
  • flex-flow는 flex-direction과 flex-wrap의 단축속성이다.
    • flex-direction는 flex item의 주축을 설정한다
      • 값으로는 row, row-reverse, column, column-reverse가 있다.
      • flex-direction이 row이면 justify-content는 수평정렬, align-items는 수직정렬 반대로 column이면 justify-content가 수직정렬, align-items가 수평정렬이 된다.
    • flex-wrap는 flex item의 줄바꿈을 설정한다.
      • 값으로는 nowrap: 한 줄에 표시, wrap: 줄바꿈 허용(item을 여러 줄로 묶음), wrap-reverse: item을 wrap의 역 방향으로 여러 줄로 묶음 이 있다.
  • Example: 모던브라우저에서 수평에서 센터로 만드는방법
  • justify-content에 들어가는 값들

float

  • float는 구형 브라우저도 지원한다. float MDN
  • float는 line box를 부모요소크기(padding을 제외한)로 하나 만든다. 그 기준으로 left던 right이던 정렬한다.
    • line box 범위를 벗어나면 float: left 나 right를 해도 그 밑으로 내려간다.
  • float를 해제하는 방법
    • 1번: 일때

Miscellaneous information

  • html {}에 폰트사이즈를 정해놓고 그 아래 요소들이 rem을 사용하면 폰트사이즈 조작이 좀더 용이해짐
  • rgb 16진수 방법 #RRGGBB
  • 색상을 정할때 rgba 함수를 권장한다
  • margin: 0 auto; 나머지 좌우 길이를 반반으로 나눠서 중앙에 배치한다.
  • margin: 0 0 0 auto; 요소가 오른쪽 정렬됨
  • <img/>의 alt 속성은 그냥 이미지 자체의 개념을 적어주어라. 로고라고 logo라고 적지말것.
  • VS code shortcuts:
    • cmd + opt + a: 약어로 개별줄 wrapping
    • cmd + opt + m: 수식평가
    • opt + 방향키: 그 줄의 text를 위아래 방향으로 옮기기
  • position: absolute;
    • float요소처럼 다른 요소위에 떠있다.
    • 자기 가장 가까운 상위요소에 의해 top, left, right, bottom offset을 정한다. (상위요소의 position이 static이 아니여야만함)
  • position: relative;
    • normal flow를 유지함
  • a tag의 color는 webkit한테 상속받은 user agent stylesheet의 영향을 받는다
    • color: inherit로 해결가능

TIL 20/10/19 HTML/CSS 첫 강의

내가 오늘 배운 것들

오늘은 패스트캠퍼스 프론트엔드 개발 스쿨 18기 과정에서 HTML/CSS 첫강의를 들었다. 강사님이 처음이라 HTML에 대한 전반적인 개념과 역사에 대해 설명해주셨고 HTML 구조 잡는법, naming convention과 Visual studio code의 extension 설치, 단축키등을 배워 보았다.
첫주 동안은 자습만 해서 지루했었는데, 이번주부터 실강을 들으니 흥미가 더욱 생기고 집중이 잘되었다. 하루하루 빠지지 않고 복습을 열심히 해야겠다.

Introduction to HTML5

  • HTML5가 제일 최신표준
  • XHTML 1.0이 XHTML에서 표준
  • HTML 4.01도 표준
  • XHTML이 더 엄격한 룰을 가지고 있음
  • HTML5는 Mark up과 API를 합친것
  • HTML5

HTML’s API

  • Web storage: 브라우저에 데이터를 저장하기 위한 공간
  • Web SQL Database/Indexed Database: 클라이언트에서 관리되는 데이타베이스를 제어
  • Application Cache: 웹 애플리케이션을 오프라인에서 사용하는 데 필요한 리소스 (HTML, CSS, JS, 이미지 등)를 클라이언트 쪽에 캐시하기 위한 기능
  • Web Workers: 메인 스레드(UI)와 독립적인 백그라운드 프로세 스로 처리되는 스크립트
  • Web Socket: 웹 브라우저와 서버 간 양방향 전이중 통신(Full
    Duplex)을 구현한 Web Socket 프로토콜을 이용할 수 있는 API
  • Notifications API: 운영 체제에 독립적인 플랫폼 수준의 알림
    메시지를 보여주는 API
  • File API(Desktop Drag-In, Desktop Drag-Out): 로컬 파일 시스템과 웹 브라우저 간에 파일을 주고받을 수 있음.
  • Geolocation: 웹 브라우저가 설치되어 있는 단말기(데스크톱,
    스마트폰 등)의 GPS 장치를 구동하여 현재 위치 정보를 받아오
    는 기능을 정의한 API
  • Device Orientation: 단말기의 센서를 이용하여 현재 방향과
    기울기가 같은 정보를 구할 수 있는 API
  • Speech Input: 단말기의 마이크로폰을 이용하여 음성을 입력
    받은 후 이를 문자로 바꿔주는 새로운 입력 방식임
  • Drag & Drop: HTML4.01에서는 Drag & Drop을 구현하기 위해 직접 자바스크립트로 개발하거나 UI 관련 자바스크립트 라이브러리를 이용해
    야 했지만, HTML5에서는 Drag & Drop API를 이용하여 쉽게 구
    현할 수 있음

Web Structure

  • 3단 구조 헤더(내비게이션) - 콘텐츠 - 푸터
  • 4단구조 헤더 - 내비게이션 - 콘텐츠 - 푸터
  • 논리적인 순서로 짤것
  • 시맨틱 마크업: 가장 기계가 이해하기 쉽게 알려주는 것, 기계처리가 가능한 것, 사람도 이해할 수 있는것
  • BEM naming convention
    • __: 안에 포함되어있다, ~의 일부분이다
    • –: ~의 상태
    • -: 일반적인 작명, Kebab case

General information about HTML5

  • header랑 footer는 여러번 사용될 수 있다.
  • main은 한번만 사용 가능하다.
  • grid는 컨텐츠 배치에 유용하다 하지만 한국에서는 IE11을 많이 쓰기 때문에 grid를 많이 안쓰는편

  • Sans-serif: 둥글둥글한 모양
  • Noto sans: 본고딕
