TIL 20/10/16 Bash 명령어, Vim, Git의 기초내용

내가 오늘 배운 것들

Bash(On mac, it’s zsh) commands

  • $ ls list segment 디렉토리내의 파일들을 보여준다.
  • $ ls -a a 플래그가 붙으면 숨김폴더도 다보여준다.
  • $ ls -l 파일들의 상세정보까지 보여준다.
  • $ mkdir {만들 디렉토리의 이름} 디렉토리(폴더) 생성
  • $ cd {디렉토리이름} 체인지 디렉토리 디렉토리로의 이동을 한다.
  • $ touch {생성할 파일의 이름} 새로운 파일을 생성한다.
  • $ pwd Print Work Directory 내가 현재 있는 절대경로를 표시해준다.
  • $ mv {파일이름} {보낼위치} 파일을 보낼위치로 옮긴다.
  • $ cp {파일이름} {위치} 파일을 그위치로 복사해준다.
  • $ mv {파일이름} {파일이름} 파일 이름을 바꿔준다. (파일 포맷도 바꿀 수 있음)
  • $ rm {파일이름} 파일을 삭제해준다.
  • $ rm -r r 플래그는 recursive로서 디렉토리안에 오브젝트까지 삭제하고 디렉토리도 삭제 (디렉토리 삭제할때 사용)
  • $ ^c control + c 지금 하고 있는 작업 실행 중단
  • $ chmod {옵션} {8진수} {파일명} rwx의 권한 변경 (r = read, w = write, x = execute)
  • 알아두면 좋을팁: pptx, db, hwp와 같은 파일 (어플리케이션을 활용하여 만드는 파일들)은 $ touch로 만들어도 제대로 동작하지 않음.
  • asterisk (*) <- 모든을 의미함 (example. $ rm server.* <- server로 시작하는 파일의 모든포맷을 삭제)

Vim simple commands

  • $ vi {edit 하고 싶은 파일명} edit하고 싶은 파일을 vim editor로 열어준다.
  • vim 처음 open하면 normal mode
  • normal mode에서 i 누르면 insert mode로서 editor안에 edit시작 가능.
  • normal mode에서 o 누르면 open new line (다음 줄에 글쓰기), 대문자 O를 사용하면 윗줄에 글쓰기
  • normal mode에서 h,j,k,l은 커서 방향이동을 의미, 순서대로 왼쪽 아래 위 오른쪽 방향 순서

Simple git commands and some tips

  • $ git config --global user.name "{github username}” 유저네임을 git configuration에 설정 해준다.
  • $ git config --global user.email "{github email address}" 이메일주소 (아이디)를 git configuration에 설정 해준다.
  • $ git config --global core.editor "vim" 에디터 설정을 git configuration에 저장해준다.
  • $ git status git repo인지 알아보는 방법, status도 알려준다.
  • .git 파일을 지우면 원래대로 돌아온다
  • $ git remote add origin {github주소} 로컬 repo에서 remote repo로 연결을 한다.
  • $ git remote -v github주소 보여줌
  • $ git add 파일이름 스테이지로 옮겨줌
  • $ git commit 스테이지에 옮겨진 파일들은 로컬 repo에 기록해준다.
  • $ git push origin master mater branch로 local repo에 있는 내용들을 remote repo로 push해준다.

How to start with git clone

  1. GitHub repo에서 초록색 Code 버튼을 클릭한다.
  2. https 주소를 옆에 서류문서 아이콘을 클릭해서 복사한다.
  3. bash로 돌아와서 git clone을 하고싶은 폴더로 이동한다.
  4. $ git clone https://복사한주소를 입력한다.
