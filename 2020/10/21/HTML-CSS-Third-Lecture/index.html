TIL 20/10/21 HTML/CSS 세번째 강의

내가 오늘 배우고 느낀점

오늘은 패스트캠퍼스에서 html/css 세번째 강의 시간 이였다. 오늘은 웹카페 클론 실습중 header에 navigation에 메인메뉴를 만들어 정렬하는 것을 배웠다. inline-block, block, flex에 대해도 배웠는데 아직 개념정리가 잘 안된 것 같아 더 구글링해서 찾아봐야겠다. 그리고 주말에 사전온라인강의도 다시들어보고 노트테이킹을 열심히 해놔서 개념을 머리 속에 굳혀놔야겠다.

button & a tag

  • <button></button>은 inline block
  • <button></button>의 box-sizing의 default값은 border-box이다.
  • <button></button>이 눌려졌는지 안눌렸는지 장애인들을 위한 알려주는 속성은 aria-pressed, 팝업이 있는지 없는지 aria-popup, 팝업이 펼쳐졌는지 안펼쳐졌는지는 aria-expanded
  • <button></button>의 default type은 submit이다.
  • <a></a> tag의 box-sizing의 default 값은 content-box이다.
  • <a></a> tag를 버튼으로 사용하려면 role="button"이라는 프로퍼티를 꼭 넣어주자. 왜냐하면 일반적으로 <a></a> tag는 클릭시 어디로 이동할거라고 예상하는데 버튼을 명시해줌으로써 클릭됬을때 이벤트가 일어날거라고 알려줄 수 있기 때문.

ETC

  • <nav></nav>는 주요메뉴나 주요링크를 포함하는 태그
  • <nav></nav>의 제목: 구조상으로는 필요하지만 보여지지 않아야 하는것 (class이름을 보통 readable-hidden, a11y-hidden, off-screen등으로 한다.)

  • 요소들을 키보드 tab버튼으로 접근하려면 <a></a>거나 <form></form>이거나 속성값으로 tableindex="0"를 줘야한다.
  • git push origin <your_branch_name> -f는 git push를 강제로 한다. 보통 GitHub에서 내용을 수정하고 pull을 안하고 로컬레포에서 수정을 하고 push를 할때 오류가 나타나는데 -f를 쓰면 강제로 GitHub remote repo에 있는 내용을 덮어쓴다.
  • 요소의 위쪽에 선을 그리는 방법 3가지:
    • background: linear-gradient 함수
    • box-shadow
    • border
  • border: 속성중 currentColor를 사용하면 본문의 폰트 색깔을 가져온다.
  • 윤곽체를 쓰려면 text-shadow를 사용할 수 있다.

틀리고 배운점

  • 상속이 되는 css 프로퍼티:
    • font
    • color
    • text-align
    • visibility
    • opacity
    • line-height
    • white-space
  • .a11y-hidden
    • 구현하는 코드:
    • .a11y-hidden을 처리해주는 이유: 구조적으로는 필요한 요소가 시각적으로는 표현될 필요가 없는 경우에 사용한다. 예전에는 position: absolute;를 주고 top: -9999px을 줘서 위로 날려버렸는데, 스크린리더를 사용하는 저시력자들은 갑자기 화면이 위쪽으로 이동되는 경험을 해서 .ally-hidden을 사용해서 구조적으로는 필요한 곳에 위치하게 하되 시각적으로는 지워주는 방식을 요즘은 사용한다.
  • position: absolute;는 position값이 static이 아닌 가장 가까운 상위요소의 위치에 배치하게 된다. 단, 조상의 position값이 모두 기본값인 static이라면, 원래 normal flow에서 가지는 위치에 위치하게된다.
