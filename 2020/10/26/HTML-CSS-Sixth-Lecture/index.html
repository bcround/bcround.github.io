TIL 20/10/26 HTML/CSS 여섯번째 강의

Posted on 2020-10-26 In dev , writing

내가 오늘 배우고 느낀점

오늘은 메인콘텐츠 안에 첫번째 그룹 맨아래 웹관련용어섹션 CSS 디자인 그리고 두번째 그룹에 검색창 HTML 구조설계와 CSS 디자인에 대해 수업 하였다. float에 대해서 어느정도 알고 있다고 생각했는데 막상 CSS 디자인을 해보니 막히는 부분이 많았었다 그래서 배운점도 많았다. 요즘도 아침에 수업시간보다 한시간씩 일찍 나오고 있는데 아직까지는 피곤하지만 할만한것 같다. 오늘의 할 일을 미루지 말고 이번 한주도 열심히 해보자.

Newly learned about display:flex

  • display: flex처리를 하면 margin 병합이 일어나지 않는다.
  • <fieldset></fieldset>이나 <form></form>에서 display: flex;를 해도 flex-item (그 안에있는 자식들)이 수평정렬이 안되는 이유:
    • <fieldset></fieldset>이나 <form></form>은 display 값이 잘 적용되지 않는 버그가 구글 크롬에 존재한다. 이를 해결하기 위해서는 <fieldset></fieldset>이나 <form></form>이 묶고 있는 요소들을 다시 새로운 <div></div>로 묶어주고 display: flex;값을 적용시켜줘야된다.

image baseline

  • <img/>요소를 block요소 안에 넣을 경우 하단에 공백이 생긴다. 이는 <img/>가 inline 요소이기 때문이다. inline 요소는 block 요소와 다르게 가상의 기준선이 존재하는데 vertical-align의 baseline에 위치하게된다. 이때 g 나 p 나 q 같은 아래삐침이 있는 글자를 고려하여 공간을 확보해주기 위해baseline 밑으로 공백이 생긴다.
  • 해결방법:
    • inline 요소를 display: block으로 바꿔준다.
    • 이미지의 vertical-align을 bottom값으로 준다.

ETC

  • float이나 position: absolute;가 적용된 요소에 width 값을 정해주지 않으면 그 요소크기만큼만 width값을 가진다. (width의 default값이 auto이니까)
  • margin: 0;으로 margin값을 재정의해줘도 되는데 예를들어 margin의 위 아래부분만 재정의 하고싶다고 하면 margin-top: 0; margin-bottom: 0;을 쓰는게 성능적 측면에서 더 효율적이다.
  • background: #000 linear-gradient(#000, #000);
    • background에서 linear-gradient 앞에 #000을 한번 더넣어준 이유는 linear-gradient가 작동하지 않는 웹브라우저가 있을 시 배경 색깔이 없으면 표시가 안되니까 linear-gradient앞에 색을 지정해줘서 색을 표시하게한다.

퀴즈에서 배운점

  • <form></form>요소는 자식 요소로 또 다른 form 요소를 포함할 수 없다.
  • <form></form>요소에도 autocomplete 어트리뷰트가 있다. 사용하면 form 전체안의 모든 서식에서 이전에 입력한 값으로 자동완성 기능을 사용할 수 있다.
  • <ul></ul>에 적용된 기본 스타일 (user-agent style)에는 다음과 같은 항목들이 있다:
    • padding-left: 40px;
    • margin: 1rem 0;
    • display: block;
    • list-style-type: disc;
